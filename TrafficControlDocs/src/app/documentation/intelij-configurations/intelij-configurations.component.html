IntelliJ Configurations

Purpose of this Document

To debug a certain subsystem you may want to run two or more subsystems at the same time. This document shows you how to setup Run-Configurations in IntelliJ to automate the building and startup of multiple subsystems.

Setup

You will probably start with a UI similar to this:
1) We start by adding the mvn package commands for each subsystem to the Run-Configurations
1.1) Click on "Run" at the top of IntelliJ. The click on "Edit Configurations".
1.2) On the window poping up, click the "+" button and select "Maven".
In the "Name"-field, insert an appropriate name. For "Working directory" select the subsystem you want to build. In "Command line", insert "package".
Repeat from step 1.2) for each subsystem.
2) Now we add a "Compound"-Configuration to package every subsystem at once.
2.1) Again, click on the "+"-button, but this time, select "Compound".
2.2) Click the "+"-button in the Compound-window and add all Maven-Configurations for each subsystem.
2.3) Click "OK" and execute the "Compound" configuration
Now Maven will build the "JAR"-files for each subsystem. The next steps will be to create Run-configurations for those "JAR"-files and another Compound-configuration to execute all at once.
3) Again, open the configuration menu. (See 1.1)
3.1) Click on the "+"-button and select "JAR-Application" (not "Application")
3.2) In the "Name" field, insert an appropriate name. In the "Path to JAR" field, click on the folder icon and select the JAR-file of the subsystem you want to setup the configuration for.
Repeat from step 3.1) for each subsystem.
4) Create another Compound-configuration in which you add the created JAR-Application-configurations
By executing this Compound-configuration you can start all the subsystems at once.
By executing a JAR-Application-configuration you can start a specific subsystems.
